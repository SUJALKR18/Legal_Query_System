# LexQueryia - AI-Powered Legal Query System

An advanced Retrieval-Augmented Generation (RAG) system for precise legal information retrieval. Answers situation-specific legal questions by citing exact law sections, act numbers, and clause references from a curated corpus of Indian statutory acts.

## 🎯 Features

✅ **Precise Legal Citations** - Responses cite exact act names, section numbers, and clause text  
✅ **Multi-turn Conversations** - Maintain session history for nuanced legal clarifications  
✅ **Multiple LLM Support** - Groq, Google AI, Anthropic, with local fallback  
✅ **Dense Semantic Search** - ChromaDB with sentence-transformer embeddings  
✅ **Full Traceability** - Every answer includes full cited clause text for verification  
✅ **Professional Disclaimers** - Clear communication of AI limitations  
✅ **React Frontend** - Modern UI with session management and markdown rendering

---

## 📋 Supported Domains

- 🔒 **Criminal Law** - IPC (1860) & Bharatiya Nyaya Sanhita (2023)
- 📋 **Right to Information** - RTI Act (2005)
- 🛒 **Consumer Protection** - Consumer Protection Act (2019)
- 👷 **Labour & Employment** - Industrial Disputes Act, Factories Act, Minimum Wages Act, Gratuity Act

**Corpus:** 61 precise legal provisions across 9 Indian statutory acts

## 🚀 Quick Start

### Prerequisites

- Python 3.9+
- Node.js 18+
- Git

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/yourusername/legal-query-system.git
   cd legal-query-system
   ```

2. **Set up the backend:**
   ```bash
   cd backend
   python -m venv venv
   # On Windows
   venv\Scripts\activate
   # On macOS/Linux
   source venv/bin/activate
   
   pip install -r requirements.txt
   ```

3. **Configure environment variables:**
   ```bash
   cp .env.example .env
   # Edit .env and add your API keys:
   # - GROQ_API_KEY (free tier available)
   # - ANTHROPIC_API_KEY (optional)
   # - GOOGLE_AI_API_KEY (optional)
   ```

4. **Ingest the legal corpus:**
   ```bash
   python ingest.py
   # This loads 61 provisions into ChromaDB
   ```

5. **Start the backend server:**
   ```bash
   python main.py
   # Server runs on http://localhost:8000
   ```

6. **Set up the frontend (in a new terminal):**
   ```bash
   cd frontend
   npm install
   npm run dev
   # Frontend runs on http://localhost:5173
   ```

7. **Access the application:**
   Open your browser and navigate to `http://localhost:5173`

## 🏗️ Architecture

### Backend Stack
- **Framework:** FastAPI 0.115.0
- **Vector Store:** ChromaDB 0.5.5 (Persistent)
- **Embeddings:** sentence-transformers (all-MiniLM-L6-v2)
- **LLM Providers:** Groq, Google AI, Anthropic
- **Server:** Uvicorn

### Frontend Stack
- **Framework:** React 19.1.0
- **Build Tool:** Vite 6.3.5
- **UI Enhancements:** Framer Motion, Lucide React
- **Markdown:** react-markdown

### RAG Pipeline
```
User Query
    ↓
Semantic Search (ChromaDB)
    ↓
Retrieve Top-5 Relevant Provisions
    ↓
Build Prompt with Retrieved Context
    ↓
Call LLM (with fallback chain)
    ↓
Extract Citations from Retrieved Provisions
    ↓
Return Answer + Citations + Disclaimer
```

## 🔧 Configuration

### Environment Variables

Create a `.env` file in the `backend/` directory:

```env
# Vector Store
CHROMA_PERSIST_DIR=./chroma_db
COLLECTION_NAME=indian_laws

# LLM Configuration (choose one or more)
GROQ_API_KEY=your_groq_api_key_here
GROQ_MODEL=llama-3.3-70b-versatile

GOOGLE_AI_API_KEY=your_google_ai_key_here
GOOGLE_AI_MODEL=gemini-2.0-flash

ANTHROPIC_API_KEY=your_anthropic_key_here
CLAUDE_MODEL=claude-sonnet-4-20250514

# Server
HOST=0.0.0.0
PORT=8000
```

### Get API Keys

- **Groq** (Recommended - Free, Fast): https://console.groq.com/
- **Google AI Studio**: https://makersuite.google.com/
- **Anthropic**: https://console.anthropic.com/

## 🔒 Security Considerations

⚠️ **Important:** Never commit `.env` files with real API keys to public repositories.

### Best Practices

1. Keep API keys in `.env` (which is in `.gitignore`)
2. Use `.env.example` as a template for contributors
3. Enable CORS only for trusted origins in production
4. Implement rate limiting for API endpoints
5. Use HTTPS in production

## 🐛 Known Issues & Limitations

- ⚠️ Limited corpus size (61 provisions)
- ⚠️ No cross-references between acts
- ⚠️ Cannot handle case law queries (statute-only)
- ⚠️ Embedding model is general-purpose (not legal-specific)
- ⚠️ No rate limiting on API endpoints
